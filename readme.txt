代理模式学习笔记:

特点:
1.两种角色:代理人、被代理人
2.被代理人不直接执行，而是通过代理人执行
3.代理人需要拿到被代理人的引用


原理(以jdk方式为例，cglib方式类似，只是cglib是用继承方式) --customer自定义实现的动态代理就是为了体现此处：
1.拿到被代理对象的引用，然后获取它的接口
2.JDK代理重新生成一个代理类，同时实现我们给的被代理对象(SpecialRenter)实现的接口(Renter)
3.把被代理对象的引用也拿到(Agency中的target就是用来存这个引用)
4.重新动态生成一个class字节码
5.编译放入jvm

一句话总结:其实就是重组了字节码


使用场景:
第三方的sdk代码无法修改，但是我们又需要重新封装，以便适应自身的业务


测试方式:
Main.java  中的main方法
